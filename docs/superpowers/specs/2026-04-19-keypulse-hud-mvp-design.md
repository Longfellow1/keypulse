# KeyPulse 状态栏 HUD 与监视器 MVP 设计

## 目标

给 KeyPulse 增加一个常驻 macOS 状态栏入口和一个独立监视器面板，让用户随时知道：

1. 今天抓到了什么值得看的内容
2. 系统现在是否正常工作
3. 用户如何立刻干预当天的汇总视角

HUD 的职责是：

- 快速操作
- 打开监视器
- 调整当天视角

监视器的职责是：

- 简约 BI 展示
- 当天汇总控制
- 重点内容处理

Obsidian 仍承担深度阅读和知识编辑。

## 产品定位

整体结构是两层：

1. `状态栏快捷层`
2. `监视器面板`

状态栏负责快操作，监视器负责看板和控制。

它不是：

- 第二个 Obsidian
- 完整桌面客户端
- 大而全设置中心

它应该做到一件事：

用户点开后，`3-5 秒` 内看懂今天最重要的内容，并能做一个操作。

## 第一层：状态栏快捷层

点击 icon 后，先弹出一个短面板，不直接堆完整信息。

### 1. 第一行：主操作

展示：

- `打开监视器`
- `保存想法`
- `标记当前窗口`
- `暂停 / 恢复`

目标：

让用户一秒内做动作。

### 2. 第二行：状态摘要

展示：

- 当前模式：`标准模式 / 专注模式 / 高敏模式 / 回顾模式`
- 当前状态：`正在采集 / 已暂停 / 需要权限`
- 今日一句话摘要
- 小型健康点：`当前看到的正文 / 屏幕识别补充 / 键入整理片段`

### 3. 第三行：今日意图

这是本次新增的 MVP 核心项。

形式：

- 一行输入入口或按钮触发输入
- 文案：`今天你最想让系统帮你盯什么？`

示例：

- `今天重点看产品决策，不放大执行噪音`
- `今天主要推进融资材料，优先保留结论和论据`
- `今天只关注模型路由和成本控制`

边界：

- 它影响的是 `采集后的汇总、排序、摘要和主题解释`
- 它不影响底层采集、隐私策略和事实存储

也就是：

`今日意图 = 当天汇总 hint，不是采集 filter`

## 第二层：监视器面板

监视器不是菜单延长线，而是一个独立面板。

布局固定成三栏：

- 左栏：概览
- 中栏：重点流
- 右栏：控制与偏好

### 1. 左栏：今日概览

展示：

- 今日有效片段
- 已过滤噪音
- 候选主题
- 已手动标记
- 最近同步时间

附加两条小趋势：

- 正文采集占比
- 噪音过滤占比

### 2. 中栏：今日重点流

展示 3 条最值得看的内容。

每条包含：

- 标题
- 来源：`当前看到的正文 / 屏幕识别补充 / 键入整理片段 / 手动保存 / 剪贴板`
- 为什么重要
- 操作：`打开详情 / 标记重要`

不做长列表，超过 3-5 条就失去 HUD 意义。

### 3. 右栏：控制与偏好

分成三块：

- 今日意图
- 额外关注事项
- 模式与快捷动作

### 4. 额外关注事项

这是长期偏好，不是当日意图。

作用：

- 提高相关候选内容排序
- 提高 HUD 中的高亮概率
- 影响主题聚合时的优先级

### 5. 快捷动作

保留以下动作：

- `保存想法`
- `标记当前窗口为重点`
- `暂停 30 分钟`
- `切换模式`
- `打开今日工作台`
- `打开系统健康检查`

## 模式设计

只保留 4 个模式：

- `标准模式`
- `专注模式`
- `高敏模式`
- `回顾模式`

说明：

- 标准模式：正常采集、正常同步
- 专注模式：弱提示、少打扰
- 高敏模式：更少正文采集、更严格隐私
- 回顾模式：强调总结和查看，不强调实时采集

## 数据来源

状态栏和监视器都不造数据，只读现有链路：

- 顶部摘要：来自 pipeline 的日摘要
- 今日重点：来自 surface snapshot 的候选排序
- 今日概览：来自 snapshot + stats
- 今日意图：来自当日 state
- 额外关注事项：来自 feedback/theme profile 或配置
- 健康状态：来自 doctor/status 和 watcher health

## 与 Obsidian 的边界

状态栏负责：

- 快速动作
- 打开监视器
- 当日意图入口

监视器负责：

- 看板阅读
- 指标概览
- 重点处理
- 当日意图和长期偏好控制

Obsidian 负责：

- 深阅读
- 整理
- 主题沉淀
- 周期复盘

## MVP 交互流

用户点击状态栏图标后：

1. 看到快捷层
2. 直接点击 `打开监视器`
3. 在监视器中查看今天的 BI 概览和重点流
4. 如有需要：
   - 设置今日意图
   - 标记重点
   - 新增关注事项
   - 切换模式

## 第一版不做

- 复杂设置页
- 时间线回放
- 富文本编辑
- HUD 内主题管理
- 长列表通知中心

## 成功标准

第一版 HUD 上线后，用户应能在 `5 秒内` 回答：

1. 今天系统有没有抓到值得看的内容
2. 当前采集是不是正常
3. 我今天最关心的事有没有被系统纳入汇总视角
4. 如果系统抓偏了，我现在可以做什么
